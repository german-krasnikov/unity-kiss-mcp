# T2 — CompilationPipeline.RequestScriptCompilation & forced compilation

## (a) Signatures & availability

- [U2019.3–U6000.3 | HIGH | https://docs.unity3d.com/2019.3/Documentation/ScriptReference/Compilation.CompilationPipeline.RequestScriptCompilation.html] `public static void RequestScriptCompilation();` exists from 2019.3; the 2019.2 docs page returns 404 → API introduced in 2019.3.
- [U2019.3–U2020.3 | HIGH | https://docs.unity3d.com/2020.3/Documentation/ScriptReference/Compilation.CompilationPipeline.RequestScriptCompilation.html] In 2019.3–2020.3 only the parameterless overload exists; no options overload is documented in 2020.3.
- [U2021.1–U6000.3 | HIGH | https://docs.unity3d.com/2021.1/Documentation/ScriptReference/Compilation.RequestScriptCompilationOptions.html] `RequestScriptCompilationOptions` enum (`None`, `CleanBuildCache`) is documented in 2021.1; its 2020.3 page 404s → the options overload was introduced in 2021.1.
- [U6000.0–U6000.3 | HIGH | https://docs.unity3d.com/6000.0/Documentation/ScriptReference/Compilation.CompilationPipeline.RequestScriptCompilation.html] Unity 6 documents both overloads: `RequestScriptCompilation()` and `RequestScriptCompilation(RequestScriptCompilationOptions options)`; options "specify whether the Editor should clear the build cache before compilation".
- [U2021.1–U6000.4 | HIGH | https://docs.unity3d.com/ScriptReference/Compilation.RequestScriptCompilationOptions.CleanBuildCache.html] `CleanBuildCache` doc text: "Clear all cached build results, resulting in a full rebuild of all scripts."
- [U2021.1–U6000.3 | HIGH | https://github.com/Unity-Technologies/UnityCsReference/blob/master/Editor/Mono/Scripting/ScriptCompilation/CompilationPipeline.cs] Source: parameterless overload forwards to `RequestScriptCompilation(RequestScriptCompilationOptions.None)`; both call `EditorCompilationInterface.Instance.RequestScriptCompilation("Requested through public api", options)`.

## (b) No-dirty behavior, incremental vs full, un-imported edits

- [U2019.3–U2020.3 | HIGH | https://docs.unity3d.com/2019.3/Documentation/ScriptReference/Compilation.CompilationPipeline.RequestScriptCompilation.html] Old semantics: makes "the Unity Editor recompile every Script in your Project asynchronously" — i.e., always a full recompile, even with nothing dirty.
- [U2021.1–U6000.3 | HIGH | https://docs.unity3d.com/2021.3/Documentation/ScriptReference/Compilation.RequestScriptCompilationOptions.html] New semantics from 2021.1: default `None` = "Unity will only recompile scripts that have changed, or that need recompiling due to changes in settings that affect script compilation."
- [U6000.0–U6000.3 | HIGH | https://docs.unity3d.com/6000.3/Documentation/ScriptReference/Compilation.CompilationPipeline.RequestScriptCompilation.html] Unity 6 text: Editor "checks whether there have been any changes to scripts, or to settings affecting script compilation, and recompiles those scripts which require it"; pass `CleanBuildCache` "to force recompilation of all scripts, even if there are no changes"; listed use cases: re-see all warnings, untracked compile inputs (response files), suspected pipeline bug.
- [U2022.3–U6000.3 | HIGH | https://docs.unity3d.com/2022.3/Documentation/ScriptReference/Compilation.CompilationPipeline-assemblyCompilationNotRequired.html] `assemblyCompilationNotRequired` event fires "when an assembly does not require compilation" — the observable signal that a Request was a no-op; page exists in 2022.3, 404s in 2021.3 (introduced in 2022.x).
- [U2021.1–U6000.3 | HIGH | https://github.com/Unity-Technologies/UnityCsReference/blob/master/Editor/Mono/Scripting/ScriptCompilation/EditorCompilation.cs] Source: `EditorCompilation.RequestScriptCompilation` only logs the reason, dirties `PrecompiledAssemblyProvider`, and cancels the active build — it does NOT call `AssetDatabase.Refresh`, import assets, or scan disk.
- [U6000.0–U6000.3 | HIGH | https://docs.unity3d.com/6000.0/Documentation/Manual/AssetDatabaseRefreshing.html] Importing+compiling on-disk .cs changes is the refresh's job: refresh will "Import and compile code-related files such as .dll, .asmdef, .asmref, .rsp, and .cs files" and then "Reload the scripting domain, if Refresh was not invoked from a script."
- [U2021.3–U6000.3 | MED | https://github.com/Unity-Technologies/UnityCsReference/blob/master/Editor/Mono/Scripting/ScriptCompilation/EditorCompilation.cs] Combined inference: **Request alone does NOT see un-imported/externally-edited .cs files** — those enter the pipeline only via asset import (Refresh/auto-refresh); Request recompiles from Unity's already-imported view.
- [U6000.0–U6000.4 | HIGH | https://docs.unity3d.com/6000.4/Documentation/ScriptReference/AssetDatabase.Refresh.html] `AssetDatabase.Refresh` imports modified/added assets synchronously but schedules script compilation asynchronously — so after Refresh, compilation of the new .cs is already queued without any Request call.

## (c) Domain reload after Request

- [U6000.0–U6000.3 | HIGH | https://docs.unity3d.com/6000.0/Documentation/ScriptReference/Compilation.CompilationPipeline.RequestScriptCompilation.html] "When compilation is successful, the Unity Editor reloads all assemblies" — reload is tied to an actual successful compile, not to the request itself.
- [U2021.1–U6000.3 | MED | https://docs.unity3d.com/2021.3/Documentation/ScriptReference/Compilation.RequestScriptCompilationOptions.html] Inference from the `None` semantics: with nothing dirty and default options, no assembly is rebuilt → no compile → **no domain reload** (Request is a silent no-op).
- [U2019.3–U2020.3 | MED | https://docs.unity3d.com/2019.3/Documentation/ScriptReference/Compilation.CompilationPipeline.RequestScriptCompilation.html] Inference: in 2019.3–2020.3 every call rebuilt all scripts, so every successful call ended in a domain reload even with zero changes.
- [U2021.1–U6000.3 | MED | https://docs.unity3d.com/6000.3/Documentation/ScriptReference/Compilation.CompilationPipeline.RequestScriptCompilation.html] With `CleanBuildCache`, docs guarantee recompilation "even if there are no changes" → a compile occurs → an assembly reload follows on success.
- [U6000.0–U6000.0 | MED | https://discussions.unity.com/t/requestscriptcompilationoptions-cleanbuildcache-not-working/1589996] Known Unity 6.0 report: `RequestScriptCompilation(CleanBuildCache)` fired `assemblyCompilationNotRequired` instead of recompiling (`assemblyCompilationFinished` never invoked); filed as IN-93874; no staff reply in thread; workaround was hand-deleting Library/Bee compile caches.
- [U2021.1–U6000.3 | HIGH | https://github.com/Unity-Technologies/UnityCsReference/blob/master/Editor/Mono/Scripting/ScriptCompilation/EditorCompilation.cs] Cost of `CleanBuildCache` (source): adds `BuildingCleanCompilation` flag and calls `ClearBeeBuildArtifacts()` (deletes incremental-build cache) → next compile is a from-scratch rebuild of every script assembly, followed by domain reload; assembly deletion happens during compilation setup (`DeleteUnusedAssemblies`), not at request time.

## (d) CompilationPipeline.codeOptimization

- [U2020.1–U6000.3 | HIGH | https://docs.unity3d.com/2020.1/Documentation/ScriptReference/Compilation.CompilationPipeline-codeOptimization.html] `public static Compilation.CodeOptimization codeOptimization` exists from 2020.1 (2019.4 page 404s); doc: "Modifying this value causes a recompile and reload of all scripts" — yes, switching forces a **full** recompile + domain reload.
- [U2020.1–U6000.3 | HIGH | https://github.com/Unity-Technologies/UnityCsReference/blob/master/Editor/Mono/Scripting/ScriptCompilation/CompilationPipeline.cs] Source: getter returns `Debug` iff script debug info is enabled; setter is a **no-op when the value is unchanged**, otherwise calls `EnableScriptDebugInfo()`/`DisableScriptDebugInfo()`; invalid value throws `ArgumentException`.
- [U6000.0–U6000.3 | HIGH | https://docs.unity3d.com/6000.0/Documentation/Manual/managed-code-debugging.html] Side effects: Debug mode = debugger attach enabled but "slower C# performance ... in Play Mode in the Editor"; Release = faster, no attach; per-session default set in Preferences > General > "Code Optimization On Startup" (mode also switchable via status-bar Debug button).
- [U2020.1–U6000.3 | MED | https://docs.unity3d.com/2020.1/Documentation/ScriptReference/Compilation.CompilationPipeline-codeOptimization.html] Automation implication (inference): toggling codeOptimization is a usable "force full recompile + reload" lever, but it mutates a user-visible editor debugging state (breaks attached-debugger workflows) — toggle-and-restore costs **two** full recompiles/reloads.

## (e) AssemblyBuilder as out-of-pipeline compile check

- [U2021.3–U6000.3 | HIGH | https://docs.unity3d.com/6000.0/Documentation/ScriptReference/Compilation.AssemblyBuilder.html] Purpose: compile scripts outside the Assets folder into a managed assembly, "uses the same references and #define directives as scripts inside the Assets folder"; configurable via `additionalDefines`, `additionalReferences`, `compilerOptions`, `flags`, `buildTarget`.
- [U2021.3–U6000.3 | HIGH | https://docs.unity3d.com/6000.0/Documentation/ScriptReference/Compilation.AssemblyBuilder.Build.html] `Build()` returns false (refuses to start) if the Editor is already compiling Assets scripts; while running it sets `EditorApplication.isCompiling = true` and shows a progress indicator; monitor via `buildStarted`/`buildFinished` events.
- [U2023.1–U6000.3 | HIGH | https://docs.unity3d.com/2023.1/Documentation/ScriptReference/Compilation.AssemblyBuilder.html] Marked **Obsolete** from 2023.1 docs onward: "This feature is being deprecated and will be removed in the future. Similar functionality can be achieved using the Roslyn compiler."
- [U2021.3–U2022.3 | HIGH | https://docs.unity3d.com/2022.3/Documentation/ScriptReference/Compilation.AssemblyBuilder.html] Still "Supported" (no deprecation notice) through the 2022.3 LTS docs line — deprecation was not backported to 2021/2022 LTS.
- [U2021.3–U6000.3 | MED | https://docs.unity3d.com/2023.1/Documentation/ScriptReference/Compilation.AssemblyBuilder.html] Verdict (inference): viable as a side-channel compile check only on 2021.3/2022.3; for a tool whose canonical target is Unity 6, do not build on it — it is deprecated, blocked whenever the editor itself compiles, and Unity's own guidance is to invoke Roslyn (csc) directly.

## DONE verdicts

- **Refresh-then-Request vs Request alone:** Request alone does not import or see on-disk edits (source-verified); `AssetDatabase.Refresh` both imports and queues compilation of changed .cs itself, so for external file edits Refresh alone is sufficient and Request adds nothing; Request's real role is recompiling without file changes (defines/settings, warnings replay, CleanBuildCache).
- **CleanBuildCache cost:** deletes Bee incremental-build artifacts → from-scratch rebuild of all script assemblies + domain reload on success; one known Unity 6.0 no-op report (IN-93874).
- **AssemblyBuilder:** usable 2021.3–2022.3, obsolete 2023.1+; mutually exclusive with editor compilation in progress; for Unity 6 tooling use an external Roslyn/csc invocation instead.
